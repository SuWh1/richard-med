## 0. Суть за 10 секунд
**Richard Med — «Aviasales для медицины».** Пациент ищет одну услугу (ОАК, приём терапевта, УЗИ)
и сразу видит реальные цены всех клиник города: с источником, свежестью, картой и маршрутом.
Сегодня для этого нужно вручную обойти десятки сайтов.

**Наша мысль:** большинство команд сделают «парсер + таблицу цен». Мы сделали
**доверенный продукт о ценах** — каждую цену можно проверить.

## 1. Как устроено (объясни простыми словами)

Система — **3 слоя**, и главный принцип: **в момент поиска мы НЕ скрапим сайты.**
Парсинг идёт отдельно (офлайн), а поиск работает только по нашей базе → быстро (<1 сек).

1. **Сбор (парсеры-адаптеры).** Офлайн обходят сайты клиник, сохраняют **сырой HTML как доказательство**,
   извлекают строки с ценами. Запуск — по расписанию (раз в сутки) или вручную из админки.
2. **Нормализация.** Сырые названия услуг приводятся к единому каталогу (**1 281 услуга**)
   через каскад: **точное → синонимы → нечёткое → семантика**. Что не сопоставилось уверенно —
   падает в **очередь на проверку**, а не показывается как ложное совпадение.
3. **Поиск + интерфейс.** Пользователь ищет → бэкенд берёт готовые активные цены из БД →
   отдаёт карточки, карту, сравнение.

### Путь одной цены (выучи — это сердце проекта)
```
сайт клиники → парсер (fetch)
   → raw_documents      (сырой HTML + хэш + дата) ← доказательство, хранится ≥90 дней
   → raw_price_items    (сырые строки: «как на сайте»)
   → нормализация       (сопоставление с каталогом + уверенность)
   → clinic_service_prices (активная цена: parsed_at, source_url, is_active)
   → поиск показывает карточку → «Паспорт цены» раскрывает весь этот путь
```

## 3. Все фичи (чтобы знать ширину продукта)

- **Поиск/интерфейс:** поиск + автодополнение, карточки «цена-герой», сортировки (Оптимальные/Дешевле/Новые), фильтры (город, категория, цена), адаптив + мобильный режим.
- **Данные/доверие:** Паспорт цены, 4 источника, сырой→нормализованный слой, бейджи свежести, очередь несопоставленных.
- **Карта/сравнение:** карта цен, синхрон. список↔карта, сравнение топ-клиник, ИИ-обзор, маршрут (ссылка 2GIS).
- **Клиника/аккаунт:** карточка клиники + филиалы (адрес, часы, телефон), 862 отзыва + рейтинги 2GIS, аналитика цен (мин/сред/макс/медиана), личный кабинет (сохранённые услуги, отслеживание цен, история поиска), вход/регистрация.
- **Бэкенд:** адаптерная архитектура (новый источник = 1 файл), конвейер с дедупликацией и логированием ошибок, 28 API-эндпоинтов, ручной + плановый запуск.
- **Качество:** TDD, 23 тест-файла, снапшот-тесты адаптеров.

## 4. Стек (если спросят «на чём»)
- **Фронтенд:** React 19, TypeScript, Tailwind v4, shadcn/ui, Leaflet + OpenStreetMap, Vite, Vercel.
- **Бэкенд:** Python, FastAPI, SQLAlchemy 2.0, Pydantic, APScheduler.
- **БД:** PostgreSQL 16 + pgvector (семантика) + pg_trgm/полнотекст (лексика), HNSW-индекс.
- **Парсинг:** httpx/requests + BeautifulSoup.
- **Сопоставление:** RapidFuzz (нечёткое) + мультиязычные эмбеддинги (семантика).
- **Гео/данные:** Yandex Geocoder (адрес→координаты, офлайн), 2GIS Public API (отзывы).

## 8. Демо-сценарий (2 мин, локально) — что делать и что говорить
> Держи темп. Если что-то грузится — переходи дальше, не молчи.

1. **Поиск «ОАК»** → автодополнение → Enter → карточки цен.
   *«Один поиск — реальные цены всех клиник. Цена — главный элемент карточки.»*
2. **Сортировка** Оптимальные → Дешевле.
   *«По умолчанию — оптимальные: цена + свежесть + расстояние. Можно переключить на самое дешёвое.»*
3. **Паспорт цены** (открыть на карточке).
   *«Вот доказательство: ссылка на источник, сырое название, дата сбора, хэш, уверенность сопоставления. Каждую цену можно проверить.»*
4. **Свежесть** (показать бейдж).
   *«Зелёный — обновлено недавно. Старше 30 дней — помечаем устаревшим и скрываем.»*
5. **Карта** (переключить) → пины с ценами → навести/кликнуть → подсветка в списке.
   *«Карта и список синхронизированы.»*
6. **Сравнение** → топ-клиники → **ИИ-обзор**.
   *«ИИ объясняет разницу клиник простым языком.»*
7. **Карточка клиники** → филиалы, часы, **отзывы 2GIS**, рейтинг.
8. **Админка → Здоровье источников** → прогоны/ошибки → запуск парсера.
   *«Операторский контроль качества данных: что собралось, где ошибка, обновить вручную.»*

**Приоритет, если время уходит:** Поиск → Паспорт цены → Карта → Сравнение/ИИ → Админка.

## 9. Вопрос-ответ жюри (готовые ответы)

- **Сколько источников?** — 4: KDL Olymp, DOQ, Invitro, Helix. Порог кейса — 3, мы перевыполнили.
- **Это легально?** — Только публичные данные, уважаем robots.txt и лимиты, без PII. **2GIS не парсим** — используем их публичное API отзывов и ссылки-маршруты.
- **Как нормализуете названия?** — Каскад: точное → синонимы → нечёткое (RapidFuzz) → семантика (эмбеддинги). Уверенность видна, сомнительное — в очередь на проверку. Никогда не выдаём ложное совпадение за точное.
- **Что если сайт клиники изменится или упадёт?** — Источники изолированы: падение одного не ломает остальные. Снапшот-тесты ловят изменение разметки. В админке видна ошибка, старые данные остаются с состаренной свежестью — продукт не «гаснет».
- **Почему поиск такой быстрый?** — Скрапинг офлайн, а поиск идёт только по нашей БД. <1 сек.
- **Откуда отзывы и рейтинги?** — 2GIS Public API, 862 реальных отзыва, рейтинги клиник.
- **Что со свежестью данных?** — У каждой цены дата сбора. Обновление ≥1 раз/сутки. Старше 30 дней — помечаем устаревшим и скрываем по умолчанию.
- **Где используется ИИ?** — Семантическое сопоставление названий + ИИ-обзор в сравнении клиник. Критичные данные (цены) — всегда проверяемы, ИИ не подменяет источник.
- **Как добавить новый источник?** — Адаптерная архитектура: новый источник = один файл по контракту (fetch/parse/clean), ядро не трогаем.
- **Как храните данные?** — Сырьё (HTML + строки) ≥90 дней, цены версионируются, доказательства не перезаписываются.
- **Бизнес-модель?** — B2C сервис доверия + B2B под капотом: аналитика и видимость для клиник, ценовой индекс для страховых и государства. Качество данных — наш ров.
- **Чем отличаетесь от конкурентов?** — Не «парсер + таблица»: Паспорт цены, честная нормализация, карта. Доверие и проверяемость — то, чего нет у генеричного скрапера.

## 10. Если что-то сломалось (план Б)

- Держи **скриншоты/видео** каждого шага как запас — если упадёт сеть/сервис, показываешь их и продолжаешь рассказ.
- Если **ИИ-обзор** не дорабатан к демо — назови его как фичу, но не кликай в сломанное состояние; покажи обычное сравнение.
